# Autogen

- Microsoft Autogen outlined in bullet points:

- Initial Release: Autogen was first released as version 0.2, focusing on basic functionalities.

- Version 0.4 Release: In January, version 0.4 was launched, representing a "from the ground rewrite" with an asynchronous, event-driven architecture. This aimed to improve observability, flexibility, and scalability in agent interactions.

- Version 0.5.1 Update: Shortly after version 0.4, Autogen 0.5.1 was released, incorporating minor updates but maintaining its overall architecture.

- Forking of the Project: Late last year, some original creators and key contributors left Microsoft to create a fork called AG2 (Agent OS 2), which is based on the earlier version (0.2) of Autogen. This split contributed to community fragmentation and confusion regarding the future of Autogen.

- Community Fragmentation: The emergence of AG2 shifted the focus away from the original Autogen, leading to uncertainty about which version users should adopt.

- Current Status: Autogen, while intriguing, is viewed as somewhat immature compared to other frameworks. It has not gained the same level of traction and community engagement as AG2 and other competitor frameworks.

## Autogen Drama

## Autogen Featrues and Components

- AutoGen Core: Event-driven framework for scalable multi-agnt AI systems.
- AutoGen AgentChat: Conversational Single and Multi-Agent Applications.
- Studio: Low-code / No code app.
- Magentic One Cli: A console-based assistant.
